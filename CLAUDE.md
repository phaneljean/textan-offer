## Project Overview

TextAnOffer is a Twilio SMS webhook service for real estate offer generation. Agents text structured offer data (e.g., "725k 3% 21day 1740 Grand Ave") and receive a filled TREC 20-19 PDF form ready for review and signing.

**Key components:**
- `app.py` — Flask web server with SMS webhook (`/sms`) and demo web form (`/demo`)
- `parser.py` — Pattern-based parser that extracts structured offer data from freeform text
- `pdf_filler.py` — PDF generation using AcroForm field filling (not coordinate overlays)

**Architecture note:** The parser uses regex patterns (no LLM calls) for common SMS formats. MLS lookup is currently stubbed — replace `lookup_mls()` in app.py with a real MLS API integration (Bridge Interactive, Spark API, etc.) before production use.

**Input validation (parser.py):**
- Price: $1 - $50M (blocks $0 or unrealistic high values)
- Down payment: 0.1% - 50% (blocks 0% or >50%)
- Closing days: 7 - 365 days (blocks same-day or multi-year closes)
- Address: minimum 5 characters after parsing

## Working with PDFs

**Critical:** The TREC 20-19 form has two versions in the wild:
- Draft/redline PDF (`20-19_draft_0.pdf`) — has ZERO fillable fields, only visual markup
- Final published PDF (`20-19_2.pdf`) — has 281 real AcroForm fields

This codebase uses **field-based filling** with `20-19_2.pdf`, not coordinate overlays. Field names are auto-generated by TREC's PDF export tool (e.g., "Texas known as" for the address field, "undefined_3" for sales price, "A The closing of the sale will be on or before" for closing date). These names are verbose but stable.

`FIELD_MAP` in `pdf_filler.py` maps logical concepts to actual field names. **Only 3 fields auto-fill from SMS data:**
- Property address
- Sales price (formatted with commas)
- Closing date (calculated from close_days)

Everything else (buyer/seller names, earnest money, option fee, financing terms) is **intentionally left blank** for the agent to complete before signing. Use `inspect_fields()` to verify field names if TREC updates the form.

## Address Validation

`app.py` includes fast, dependency-free address validation (`validate_address()`) that runs before PDF generation:

**Blocking errors (prevent PDF generation):**
- Missing street number
- Missing street suffix (St, Ave, Rd, Blvd, Dr, Ln, Ct, etc.)

**Non-blocking warnings (PDF still generates):**
- No TX or Texas ZIP code detected
- Address looks too short (< 3 words)

This is NOT full USPS/geocoding validation — it's a sanity check before legal contract generation. The validator returns `{"valid": bool, "reason": str|None, "warnings": list[str], "normalized": str}`.

## Deployment

Deploy to Railway with:
```bash
git push origin main
```

Railway will:
1. Install dependencies from `requirements.txt`
2. Start Gunicorn on `$PORT`
3. Serve SMS webhook at `/sms`
4. Serve generated PDFs at `/offers/<filename>`

## Environment Variables

- `TWILIO_AUTH_TOKEN` (optional, recommended for prod) — Twilio auth token for request validation
- `TREC_TEMPLATE_PATH` (default: `20-19_2.pdf`) — Path to the clean TREC form template
- `OFFER_OUTPUT_DIR` (default: `generated_offers`) — Directory to store generated PDFs
- `PORT` (default: `5000`) — Server port for Flask/Gunicorn

## Code Organization

The codebase has duplicate files in root and `textan-offer/` subdirectory. The root-level files are the current working versions. The `textan-offer/` directory appears to be an earlier snapshot or staging copy — verify which is canonical before making changes to both.

## Testing Strategy

Use `/demo` for end-to-end testing without SMS:
1. Visit http://localhost:5000/demo
2. Type an offer string (e.g., "725k 3% 21day 1740 Grand Ave")
3. Submit to see parsed data and generated PDF link

This bypasses Twilio entirely while exercising the same parse → validate → fill → serve logic.
